# Tarefa — Entrega Final do Projeto Tecnológico

Prezado aluno!

Nesta atividade, você deverá realizar a **entrega final do seu Projeto Tecnológico**, contemplando a implementação do sistema e a sua disponibilização para acesso e avaliação.

O objetivo desta etapa é verificar se o sistema desenvolvido reflete a proposta, atende aos requisitos e segue o projeto definido nas etapas anteriores, além de avaliar o funcionamento, a organização e a qualidade geral da solução entregue.

## Itens obrigatórios da entrega

A entrega deve conter obrigatoriamente:

- ✅ **Código-fonte** do sistema;
- ✅ **Arquivo com instruções** para execução/instalação do sistema;
- ✅ **Link público** para acesso ao sistema (ou link para download/instalação, no caso de aplicativo);
- ✅ **Documento de entrega**, contendo uma tabela com:
  - As funcionalidades propostas no projeto;
  - O status de cada uma (**Implementada**, **Parcialmente implementada** ou **Não implementada**);
  - Quando aplicável, a justificativa para funcionalidades não implementadas.

Todos os arquivos devem estar organizados em uma única pasta (nomeada com seu nome completo, sem abreviações) e enviados conforme orientações da atividade.

## ⚠️ Atenção

- A implementação deve estar coerente com a proposta, os requisitos e o projeto entregues nas etapas anteriores.
- Sem a entrega válida da proposta, dos requisitos e do projeto, esta etapa não será considerada, pois a implantação depende diretamente dessas fases.
- O sistema deve estar acessível para avaliação. Links quebrados, sistemas que não abrem ou aplicativos não instaláveis inviabilizam a correção.
- Projetos baseados apenas em plataformas prontas (como WordPress ou similares) não serão aceitos. Frameworks podem ser utilizados, desde que haja desenvolvimento próprio.
- Caso o sistema não esteja disponível para acesso ou falte algum dos itens obrigatórios, a atividade será considerada não entregue.

## 📊 Avaliação

Esta etapa vale **5,5 pontos** e será avaliada considerando:

- A aderência à proposta, aos requisitos e ao projeto;
- O funcionamento do sistema;
- A organização do código e dos arquivos;
- A qualidade da interface e da navegação;
- A implementação efetiva das funcionalidades propostas;
- A disponibilização correta do sistema para acesso público ou instalação.
